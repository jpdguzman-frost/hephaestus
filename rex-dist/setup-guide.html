Before You Start

✓

Node.js (v20 or newer)

Download from nodejs.org — click the big green LTS button and run the installer.

✓

Claude Desktop, Claude Code, Cursor, or Windsurf — any MCP-compatible client.

Quick Start (Recommended)

Open Terminal and run this single command. It will do everything for you.

Terminal
$ cd /path/to/rex-dist && bash install.sh

How to open Terminal: Press ⌘ Space, type Terminal, press Enter.

1

Install the MCP Server

If you used the Quick Start above, this is already done! Otherwise, follow these steps manually.

a
Open Terminal (press ⌘ Space, type "Terminal", press Enter)
b
Navigate to the rex-dist/server folder:
Terminal
$ cd /path/to/rex-dist/server
$ npm install --production
added 86 packages in 5s
c
Copy the server and plugin folders to a permanent home:
Terminal
$ mkdir -p ~/rex
$ cp -r server plugin ~/rex/
2

Import the Figma Plugin

This adds Rex as a development plugin in your Figma app.

a
Open the Figma Desktop app
b
Open any file, then find the plugin import menu:
Navigate to:
Figma Menu ▸ Plugins ▸ Development ▸ Import plugin from manifest...
c
In the file picker, navigate to:
~/rex/plugin/manifest.json
Tip: In the file picker, press ⌘ Shift G to open the "Go to Folder" dialog, then type ~/rex/plugin and press Enter. Select manifest.json.
d
You should see "Rex" appear under Plugins → Development. That's it!
3

Configure Your AI Client

Tell your AI client how to find the Rex server. Pick your client below:

a
Open Claude Desktop → Settings (⌘ ,) → Developer → Edit Config
b
This opens a JSON file. Replace its contents with:
{
  "mcpServers": {
    "rex": {
      "command": "node",
      "args": ["~/rex/server/index.js"]
    }
  }
}
c
Save the file (⌘ S) and restart Claude Desktop
Note: If you already have other MCP servers configured, add the "rex" entry inside your existing "mcpServers" object — don't replace the whole file.
a
Run this command in Terminal to add Rex globally:
Terminal
$ claude mcp add rex node ~/rex/server/index.js

That's it! Rex will be available in all Claude Code sessions.

Troubleshooting

"node" is not recognized / command not found +
Node.js isn't installed or isn't in your PATH.

Fix: Download and install Node.js from nodejs.org (use the LTS version). After installing, close and reopen Terminal before trying again.
Rex plugin shows "connecting" but never connects +
The MCP server isn't running. The server only starts when your AI client launches it.

Fix: Make sure your AI client (Claude Desktop, etc.) is open and has the Rex MCP config added. Try restarting the AI client. The plugin will connect automatically once the server starts.
I can't find ~/rex/plugin in the Figma file picker +
Fix: In the Figma file picker, press ⌘ Shift G to open "Go to Folder". Type ~/rex/plugin and press Enter. Then select manifest.json.
Port 7780 is already in use +
Another instance of Rex (or another app) is using port 7780.

Fix: Open Terminal and run:
lsof -i :7780 | grep LISTEN
This shows what's using the port. If it's a previous Rex instance, kill it with:
kill $(lsof -ti :7780)
Claude Desktop shows "rex" as disconnected / failed +
The path to the server file may be incorrect.

Fix: Open Terminal and verify the file exists:
ls ~/rex/server/index.js
If it says "No such file or directory", re-run the install script. Also make sure you're using the full absolute path (starting with /Users/yourname/) in the config, not ~ — some clients don't expand the tilde.

I need to use a different AI client not listed here +
Rex works with any MCP-compatible client. The config is always the same — you just need to tell the client to run:
node ~/rex/server/index.js
Check your client's documentation for how to add MCP servers.
