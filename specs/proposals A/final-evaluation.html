Part 2

Best Ideas Across Proposals

Seven individually strong and complementary ideas, regardless of which proposal they came from.

1
Proposal 05
Passive Observation via documentchange
Rex already knows every node it created. documentchange gives exact before/after values without any matching algorithm. This eliminates the node-matching problem entirely.
2
Proposal 04
Golden Templates as Primary Learning Artifact
Storing the refined SOM as a reusable template captures 100% of decisions without extraction, inference, or rule formation. Templates are facts, not abstractions.
3
Proposal 06
Adversarial Change Taxonomy
Six categories: AI error, screen-specific, artifact cleanup, reverted experiment, content-driven, genuine principle. The missing piece that prevents self-poisoning.
4
Proposal 02
Content Fingerprinting for Fallback Matching
For screens Rex didn't build, content fingerprinting provides the best fallback. Simple, deterministic, handles renames and reparenting gracefully.
5
Proposal 05, Round 2
BuildManifest for Node-to-Role Mapping
Explicit mapping of Figma node IDs to SOM roles at build time. Trivial to build (one Map.set per node), invaluable for pattern extraction.
6
Consensus (P01, P02, P05, P06)
Brand-Scoped Storage
Rules are brand-scoped by default. Cross-brand generalization only for directional patterns. Every proposal that addressed contradictions converged here.
7
Proposal 03
Screenshot-as-Index for Cold Start
Visual comparison bootstraps learning faster than raw property diffs for the first 5 screens. Once patterns exist, vision becomes unnecessary.

The Merged Solution

Template-First with Passive Pattern Learning. Two complementary tracks that operate independently but reinforce each other.

At Build Time

  1. Query for matching Golden Template (brand, screen type, tags)
  2. If template exists (score > 0.5): use template SOM, populate content slots
  3. If no template: fall back to Osiris reference + apply confirmed property patterns (confidence >= 0.7)
  4. Build using Rex tools

After Refinement

  1. Passive observer records all property changes via documentchange, mapped to roles via BuildManifest
  2. Observation window closes on idle (60s), new build, or page switch
  3. Adversarial Filter classifies each change into 6 categories
  4. Genuine preferences enter pattern store; AI errors route to builder diagnostics

Pattern Promotion

3-4

5+

Part 4

Recommendation

Build Template Evolution first, then layer Passive Observation with Adversarial Filtering.

Templates deliver value on Day 1. Passive observation fills the generalization gap. The adversarial filter prevents self-poisoning. Everything else is Phase 5.

Ships in 2 Weeks

  • Golden Templates with 100% fidelity, zero inference risk
  • Silent edit observation via documentchange
  • Adversarial filter rejects ~50% as noise
  • Recurring patterns auto-promoted to confirmed rules
  • Self-corrects when designer overrides pattern-applied values

After 3 Months

  • 20+ templates per brand covering common screen types
  • 10-15 confirmed property patterns per brand
  • 80-90% reduction in refinement time on templated screens
  • 40-60% reduction on novel screen types via patterns alone
  • The designer has forgotten the system is learning
